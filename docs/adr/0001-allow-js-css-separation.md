# ADR 0001: JS/CSS 파일 분리 허용 (단일 파일 원칙 완화)

- **상태**: Accepted
- **날짜**: 2026-04-22
- **대체 대상**: AGENTS.md "단일 파일 원칙" 조항의 일부

## 배경

기존 규칙은 **`index.html` 단일 파일**(HTML + CSS + JS 인라인) 원칙이었다. 이 원칙은 다음 이점이 있었다:

- 빌드 도구 없이 열고 바로 동작
- 배포/공유 단순 (파일 하나만 전달)
- 로컬 파일 시스템에서도 동일하게 동작

그러나 다음 한계가 명확해졌다:

- 현 시점 `index.html`이 **3,285줄**로 단일 파일의 가독성·탐색성이 한계에 이름
- 기능이 늘어날수록(예: 회의 기록 / 노트 / 대화 번역 탭) 섹션 간 경계가 흐려지고, 수정 시 관심사 섞임이 커짐
- 정적 호스팅/로컬 파일 환경에서도 여러 `.js` / `.css` 파일을 `<script>` / `<link>`로 직접 참조하는 것은 빌드 도구 없이 가능함

## 결정

**JS/CSS 파일 분리를 허용한다.** 단, **과도한 분리는 하지 않는다.**

### 허용되는 분리

- `index.html`에서 크게 덩어리진 코드 섹션을 **기능 단위**로 `.js` / `.css` 파일로 분리 가능
- HTML은 여전히 `index.html` 단일 파일 유지 (엔트리포인트는 하나)
- 외부 파일은 `<script src="...">` 또는 `<link rel="stylesheet" href="...">`로 정적 참조

### 금지 / 지양

- **번들러/빌드 시스템 도입 금지** (webpack, vite, esbuild 등) — 브라우저가 바로 로드하는 구조 유지
- **패키지 매니저(npm/yarn) 도입 금지** — 외부 의존성은 기존대로 CDN 사용
- **과도한 파일 분리 지양**:
  - "한 함수 = 한 파일" 같은 미시 분할 금지
  - 기능 단위(feature-oriented)로 묶어 분리 — 예: `translation.js`, `meeting.js`, `notes.js`
  - 10개 이상의 파일로 쪼개지 말 것 (운영상 기준)
- **모듈 시스템(ES Module import/export) 사용 조건부**: 로컬 파일(`file://`)에서도 열 수 있어야 한다면 전역 `window.*` 네임스페이스 방식을 우선 고려

### 권장 구조 (예시)

```
/
  index.html              # 엔트리 + 마크업 + 부트스트랩 <script> 태그들
  assets/
    css/
      base.css            # 토큰, 레이아웃 공통
      tabs.css            # 탭 관련 (신규)
    js/
      state.js            # 전역 state
      i18n.js             # 국제화
      translation.js      # 대화 번역
      meeting.js          # 회의 기록 (신규)
      notes.js            # 노트 (신규)
      app.js              # 초기화/오케스트레이션
```

> 위 구조는 예시다. 실제 분리 단위는 해당 작업에서 ADR 또는 커밋 설명으로 기록한다.

## 근거

- **파일 크기**: 3,285줄은 한 화면에 올라오지 않으며, 탭 추가 작업은 여기에 수백 줄을 더 얹는다. 분리하지 않으면 읽기·리뷰·수정 비용이 비선형으로 증가
- **빌드 없이도 충분**: 브라우저의 `<script>` 순차 로드만으로 기능 단위 분리가 가능 — 즉 "단일 파일 원칙"이 주던 이점(빌드 없음, 공유 단순)을 크게 훼손하지 않는 선에서 분리 가능
- **번들러/패키지 매니저 배제 유지**: 의존성 관리 복잡도와 툴체인 유지비용은 기존처럼 회피 — 여기까지 열면 프로젝트 정체성이 바뀐다

## 결과

### 긍정

- 기능 단위 수정 시 영향 범위가 명확해짐
- 리뷰/탐색/검색 용이성 개선
- 탭 도입 같은 대형 변경의 충돌 위험 감소

### 부정 / 주의

- 로드 순서 의존성이 생김 → `<script>` 태그 순서로 관리 (전역 `state` 초기화 → 각 feature → `app.js` 부트)
- `file://` 로컬 실행 시 CORS 제약으로 `fetch`/ES Module 사용에 주의 — 전역 네임스페이스 우선
- 파일 수가 늘어나면 **"과도한 분리 지양"** 원칙이 언제든 재검토될 수 있음

### 후속 조치

- `AGENTS.md`의 "단일 파일 원칙" 섹션을 본 ADR로 링크하여 완화
- 탭 도입 작업(`feature/meeting-notes-translation-tabs` 브랜치)에서 본 ADR을 근거로 실제 분리 적용
