# 시스템 오디오 버튼 UX 개선 스펙

## 문제

"시스템 오디오" 버튼만 보고는 기능의 목적을 알 수 없다. 사용자가 이 버튼이 왜 필요하고 언제 써야 하는지 이해하지 못한다.

## 해결

버튼 하단에 서브텍스트를 추가하여 기능의 목적을 즉시 전달한다.

## 변경 전후

**변경 전:**
```
🎤 시스템 오디오
```

**변경 후:**
```
🎤 시스템 오디오
   상대방/영상 소리도 번역
```

## 상세 스펙

### HTML

버튼 내부에 `<small>` 태그로 서브텍스트 추가:

```html
<button class="ctrl-btn btn-sysaudio" id="btnSysAudio" aria-label="Toggle system audio capture">
  &#127908; <span data-i18n="sysAudio">시스템 오디오</span>
  <small class="btn-desc" data-i18n="sysAudioDesc">상대방/영상 소리도 번역</small>
</button>
```

### CSS

- 버튼: `flex-direction: column` 으로 변경하여 텍스트와 서브텍스트를 세로 배치
- 서브텍스트: 폰트 크기 `0.7em`, 투명도를 낮춰 메인 텍스트와 시각적 위계 구분
- 활성/비활성 상태에 맞춰 서브텍스트 색상도 연동

### i18n

| 키 | 한국어 | 영어 |
|---|---|---|
| `sysAudioDesc` | `상대방/영상 소리도 번역` | `Translate meeting/video audio too` |

### 반응형

- 버튼 높이가 서브텍스트만큼 자연스럽게 늘어남
- 컨트롤바의 다른 버튼 레이아웃에 영향 없음 (해당 버튼만 높이 변경)

## 범위 밖

- 화면 공유 다이얼로그 전 안내 팝업: 불필요 (사용자 확인)
- 온보딩 가이드 / 튜토리얼: 불필요
- 버튼 이름 변경: 기존 "시스템 오디오" 유지

## 성공 기준

1. 버튼을 처음 보는 사용자가 서브텍스트만으로 기능 목적을 이해할 수 있다
2. 기존 컨트롤바 레이아웃이 깨지지 않는다
3. 한국어/영어 i18n이 정상 동작한다
4. 활성/비활성 상태에서 서브텍스트 스타일이 자연스럽다
